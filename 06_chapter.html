5. Integration Architecture - How They Work Together

The convergence of AG-UI and A2UI protocols creates a powerful, complementary architecture that addresses different layers of the agent-user interaction stack. Rather than competing technologies, they form an integrated ecosystem where AG-UI handles the communication pipeline and A2UI defines the interface payload format.

Architectural Layers and Responsibilities

The AG-UI + A2UI stack operates across three distinct architectural layers:

  • Transport Layer (AG-UI): Connection management, event routing, authentication
  • Protocol Layer (Both): Message formats, interaction patterns, state management
  • Presentation Layer (A2UI): Component rendering, layout, user interactions

Message Flow Integration

// Complete integration flow example // 1. User input via AG-UI {   "type": "HUMAN_MESSAGE",   "content": "Book a table for 4 tomorrow at 7pm",   "thread_id": "conversation_001" }  // 2. Agent processing and A2UI generation {   "type": "GENERATIVE_UI_UPDATE",   "payload": {     "type": "surface_initialize",     "surface_id": "booking_form",     "properties": {       "title": "Restaurant Reservation",       "layout": "vertical_flow"     }   } }  // 3. Progressive component addition {   "type": "GENERATIVE_UI_UPDATE",   "payload": {     "type": "component_add",     "component": {       "id": "party_size",       "type": "number_input",       "properties": {         "label": "Party Size",         "value": 4,         "min": 1,         "max": 12       }     }   } }  // 4. User interaction back to agent {   "type": "USER_INTERACTION",   "component_id": "submit_reservation",   "event": "click",   "data": {     "party_size": 4,     "date": "2026-01-16",     "time": "19:00",     "preferences": ["window_table"]   } }

State Synchronization Strategies

Managing state consistency between agents, applications, and generated UIs requires sophisticated synchronization strategies:

  • Client-side state management for responsive user experiences
  • Agent-driven state updates for business logic integration
  • Conflict resolution for concurrent modifications
  • Eventual consistency models for distributed scenarios
